# CrownFalle Proto — Game Loop Design Document

> 날짜: 2026-03-19
> 상태: 확정 (1차)
> 엔진: Unity 6.3 LTS + C# + TBSF (Turn Based Strategy Framework)
> 렌더링: 2D (Dust and Salt 스타일)
> 전환 배경: Godot 3D 에셋 소싱 병목 → Unity 2D 전환 확정

---

## 1. 게임 루프 개요

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Town / Camp                World Map                      │
│   (거점)          ──────→    (여행/탐험)                     │
│     ↑                            │                          │
│     │                            ▼                          │
│   Roster                     Battle                         │
│   Management     ←──────     (전투)                         │
│   (용병단 운영)                                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

4개 레이어가 순환하며 게임을 구성한다:
1. **Town / Camp** — 출격 전 준비, 귀환 후 회복, 스토리 진행점
2. **World Map** — 마을 간 이동, 랜덤 이벤트, 전투 조우, 자원 소모
3. **Battle** — 헥스 그리드 턴제 전술 전투 (기존 T1~T2 시스템 활용)
4. **Roster Management** — 용병 성장, 장비, 로스터 로테이션

---

## 2. Layer 1: Town / Camp (거점)

### 2-1. 기본 형태

- **복합 거점 구조**: 여러 마을 순회 + 이동식 캠프
  - **마을 (Town)**: 고정 위치. 각 마을마다 고유 시설/NPC/퀘스트
  - **캠프 (Camp)**: 이동식. 여행 중 아무 곳에서나 설치 가능
- 마을은 상위 시설 (치료, 모집, 상점, 퀘스트), 캠프는 기본 시설 (휴식, 간이 제작)

### 2-2. 핵심 자원 (4종)

| 자원 | 역할 | 획득처 | 소모처 |
|------|------|--------|--------|
| **Gold** (금화) | 급여, 구매, 수리 | 전투 루팅, 퀘스트 보상, 거래 | 용병 급여, 장비 구매/수리, 마을 시설 이용 |
| **Food** (식량) | 여행/캠프 소모 | 구매, 자원 발견, 사냥 | 여행 이동 시 소모, 캠프 휴식 시 소모 |
| **Morale** (사기) | 용병단 전투력/이탈 | 전투 승리, 급여 지급, 이벤트 | 전투 패배, 동료 사망, 미지급, 이벤트 |
| **Reputation** (평판) | 지역 영향력 | 퀘스트 완수, 선택지 결과 | 범죄 행위, 실패, 선택지 결과 |

### 2-3. 스토리 전달 방식

- **미확정** — TBSF 대화 시스템 확인 후 결정
- 후보: Dust and Salt식 텍스트 선택지 / NPC 대화 + 퀘스트 게시판 / 내러티브 이벤트 팝업

### 2-4. 마을 시설 (잠정)

| 시설 | 기능 | 비고 |
|------|------|------|
| 여관 (Inn) | 용병 모집, 퀘스트 게시판 | 각 마을마다 모집 풀이 다름 |
| 대장간 (Smithy) | 장비 구매, 수리, 강화 | 무기 기반 정체성과 연결 |
| 치료소 (Infirmary) | 부상 치료 | 부상은 마을에서만 회복 |
| 상점 (Market) | 식량/소모품 구매, 루팅품 판매 | — |
| 의뢰소 (Board) | 계약/퀘스트 수주 | 전투 + 보상 연결 |

### 2-5. 캠프 시설 (잠정)

| 시설 | 기능 | 비고 |
|------|------|------|
| 모닥불 (Campfire) | HP 회복, 식량 소모 | 피로도 리셋 |
| 간이 공방 (Workbench) | 기본 수리, 소모품 제작 | 대장간보다 제한적 |
| 보초 (Watch) | 야습 방지 확률 증가 | 용병 배치 필요 |

## 3. Layer 2: World Map (여행/탐험)

### 3-1. 이동 방식

- **Dust and Salt식 포인트 클릭**
- 일러스트 맵 위에 클릭 가능한 위치(Location) 표시
- 위치를 클릭하면 이동 → 이동 거리에 비례하여 자원 소모
- 이동 중간 과정 없음 (이동 = 즉시, 결과만 처리)
- 구현 비용 최소, 콘텐츠(이벤트/전투)에 집중

### 3-2. 여행 중 자원 소모

- **식량 (Food)**: 이동 거리에 비례하여 소모
- **피로도 (Fatigue)**: 이동 거리에 비례하여 누적
  - 피로도 상한 도달 시 캠프 강제 (또는 능력치 페널티)
  - 캠프 휴식으로 피로도 리셋 (식량 소모)

### 3-3. 월드맵 이벤트 (4종)

| 이벤트 | 설명 | 빈도 |
|--------|------|------|
| **적 조우** | 전투 진입 (회피 가능할 수도 있음) | 높음 |
| **랜덤 이벤트** | 텍스트 + 선택지 (사기/평판/자원 변동) | 중간 |
| **자원 발견** | 식량/금화/아이템 획득 | 중간 |
| **상인 조우** | 이동 중 거래 기회 | 낮음 |

### 3-4. 월드맵 구조 (잠정)

```
[시작 마을] ──── [숲 지역] ──── [산악 마을]
                    │                │
               [폐허/던전]     [항구 마을]
                    │                │
               [늪 지역] ──── [최종 지역]
```

- 각 지역(Region)은 고유 적 타입, 이벤트 풀, 난이도를 가짐
- 진행에 따라 새 지역 해금 (또는 자유 탐험)
- 상세 지역 구성은 스토리/월드빌딩 확정 후 결정

## 4. Layer 3: Battle (전투)

### 4-1. 기존 시스템 활용

Godot에서 구현 완료된 T1~T2 전투 시스템의 **설계 로직**을 Unity/TBSF 위에 이식한다.
코드는 새로 작성하지만, 알고리즘과 JSON 데이터는 그대로 활용.

**이식 대상 (T1 — 기반):**
- 5대 주요 스탯 (STR/DEX/CON/INT/WIL) + 2차 파생 스탯
- ARM/RES 게이지 시스템 (게이지 차감 + 소진 상태)
- 전투 공식 (HIT/CRIT/데미지 + damage_type 분기)
- STA 시스템 (행동 비용)
- 교전 시스템 v2 (기회 공격, 조건부 이탈 4종)

**이식 대상 (T2 — 확장):**
- StatusManager (11종 상태이상: CC/DoT/디버프)
- SkillManager (액티브 스킬 프레임워크)
- ElementResolver (4원소 규칙)
- VP 시스템 (사기)
- 조건부 이탈 (VP/스킬/아군 지원)

### 4-2. 전투 파라미터

| 항목 | 값 | 비고 |
|------|-----|------|
| 출격 인원 | **3~10명** | 시나리오에 따라 가변 |
| 그리드 | 헥스 (TBSF 제공) | staggered hex, TILE_SIZE는 TBSF 기본값 |
| 턴 방식 | TBSF TurnManager 활용 | 기존 설계와 매핑 필요 |
| 전투 보상 | **XP + 루팅** | 레벨업 시스템 포함 |

### 4-3. 전투 → 게임 루프 연결

```
전투 승리
  ├── XP 획득 → 레벨업 (스탯 성장, 스킬 해금)
  ├── 루팅 → 금화, 장비, 소모품
  ├── VP 결과 → 사기(Morale) 변동
  └── 부상/사망 → Roster Management로 전달

전투 패배
  ├── 용병 사망 (영구) 가능
  ├── 사기 하락
  └── 루팅 없음 (또는 감소)
```

### 4-4. HP/부상 회복 구조

| 상태 | 회복 장소 | 비용 |
|------|----------|------|
| **HP 손실** | 캠프 휴식 | 식량 소모 |
| **부상 (Injury)** | 마을 치료소만 | 금화 소모 + 시간 |
| **사망** | 회복 불가 | 영구 손실 |

- HP는 캠프에서 식량을 소모하여 회복 (전투 후 자동 회복 없음)
- 부상은 마을 치료소에서만 치료 가능 (T3 부상 시스템과 연결)
- 부상 상태로 전투 출격 가능하지만 능력치 페널티

## 5. Layer 4: Roster Management (용병단 운영)

### 5-1. 스탯 시스템

- **기존 CrownFalle 5대 스탯 유지**: STR / DEX / CON / INT / WIL
- 2차 스탯 파생 공식 유지 (HP, STA, MP, HIT, CRIT, MV 등)
- ARM/RES는 장비+패시브+버프에서 산출되는 3차 스탯
- 레벨업 시 스탯 포인트 배분 (상세 방식 미확정)

### 5-2. 용병 정체성

- **클래스 레이블 없음** (기존 CrownFalle 재설계 방향 유지)
- **무기 기반 정체성**: 장착 무기가 사용 가능한 스킬 풀을 결정
- 양손 슬롯에 무기 자유 장착 (물리/마법 혼합 포함)
- 파츠 기반 장비 (머리/갑옷/장갑/신발/악세서리)
- 스킬 습득 방식: 미확정 (추후 설계)

### 5-3. 용병 사망/손실

- **영구 사망 (Permadeath)**
- 전투 중 HP 0 + Death 조건 충족 시 영구 손실
- 사망한 용병의 장비는 회수 가능 (또는 조건부)
- 높은 사기(Morale) 유지 → 이탈 방지
- 낮은 사기 → 이탈 가능성 (사망은 아니지만 로스터에서 제거)

### 5-4. 용병 모집

- 마을 여관에서 모집 (각 마을마다 모집 풀 다름)
- 신규 용병은 레벨 1 또는 낮은 레벨로 합류
- 모집 비용: 금화
- 로스터 상한: 미확정 (10~20명 범위 예상)

### 5-5. 레벨업

- XP 기반 레벨업
- 레벨업 시 혜택:
  - 스탯 포인트 배분
  - 스킬 해금/강화 (무기 기반)
  - 성향(Trait/Disposition) 선택 (마일스톤, 상세 미확정)
