## 보강 1: "안 할 것" (Won't Do) 상태 추가

### 문제

의도적으로 포기한 항목과 완료한 항목이 구분되지 않음.

```
예: "ARM/RES % 밸런스 조정" → 게이지형 전환으로 의미 없어짐
  ✅에 넣으면? → "했다"로 오독
  삭제하면?   → "왜 안 했지?" 이력 유실
```

### 해결

TODO.md에 ❌ 섹션 추가:

```markdown
---

## ❌ 안 할 것 (Won't Do)
> 의도적 포기/폐기 — 사유 필수 기재

- [~] {항목} `[출처]` `[폐기일]` — 사유: {한 줄}
```

### 커맨드

```
/todo wontdo {키워드} — {사유}

예:
  /todo wontdo ARM % 밸런스 — 게이지형 전환으로 % 방식 폐기

출력:
  ❌ 폐기: ARM/RES % 밸런스 조정
     사유: 게이지형 전환으로 % 방식 폐기
```

### 규칙

- 사유 필수. 사유 없이 `/todo wontdo`하면 "사유를 입력해주세요" 요청
- ❌ 항목도 월 단위 아카이브 대상 (git log에 이력)
- `/todo review`에서 STALE 항목 정리 시 "삭제" 대신 `wontdo` 선택지 제공

### CLAUDE.md 추가

```markdown
- `/todo wontdo {키워드} — {사유}` — ❌ 폐기 이동 (사유 필수)
```

## 보강 2: 항목 메모/컨텍스트

### 문제

한 줄짜리 항목만 있어서 시간이 지나면 맥락을 잃음.

```
"투사체 GLB 에셋 교체" → 3주 후: "어떤 에셋? 어디서 구하기로 했지?"
```

### 해결

항목 바로 아래에 들여쓰기된 메모 라인을 허용한다.

```markdown
- [ ] 투사체 GLB 에셋 교체 `[설계:phase-b]` `[2026-03-13]`
      Quaternius에 화살/마법탄 에셋 있는지 확인. 없으면 Kenney 검색.
      관련: Projectile.gd visual.mesh_type == "custom" 분기
```

### 커맨드

```
/todo note {키워드} {메모 내용}

예:
  /todo note 투사체 GLB Quaternius에 화살 에셋 확인, 없으면 Kenney

출력:
  📝 메모 추가: 투사체 GLB 에셋 교체
     → "Quaternius에 화살 에셋 확인, 없으면 Kenney"
```

### 규칙

- 메모는 선택사항 — 모든 항목에 강제하지 않음
- 메모는 항목 바로 아래, 6칸 들여쓰기 (마크다운 리스트 하위와 구분)
- 메모가 여러 줄이면 줄마다 6칸 들여쓰기
- `/todo list` 출력 시 메모는 기본 숨김, `/todo list -v` (verbose)로 메모 포함 출력

### CLAUDE.md 추가

```markdown
- `/todo note {키워드} {메모}` — 항목에 컨텍스트 메모 추가
- `/todo list -v` — 메모 포함 상세 출력
```

## 보강 3: 에이전트 자동 발견 TODO 처리

### 문제

원주가 등록하는 경우만 규칙이 있고, Claude Code가 구현/리뷰 중 스스로 할 일을 발견하는 경우가 빠져 있음.

```
예: @Implementor가 CombatUnit.gd 수정 중 발견
  "receive_hit()에서 null 체크가 빠져 있는데, 지금 고치면 범위 밖이다"
  → 원주에게 보고? 자동 등록? 무시?
```

### 해결: 제안 후 승인 방식

에이전트는 자동 등록하지 않는다. 대신 **제안**하고 원주가 승인한다.

```
Claude Code (@Implementor):
  💡 TODO 제안: receive_hit()에 null 체크 누락 — 현재 작업 범위 밖
  등록할까요? (y/n)

원주: y

Claude Code:
  ✅ TODO 등록: receive_hit() null 체크 추가 → 🟡 다음
```

### 규칙

| 상황 | 처리 |
|------|------|
| 현재 작업 범위 **안** | 즉시 수정 (TODO 불필요) |
| 현재 작업 범위 **밖** + 사소함 | 제안 → 원주 승인 → 🟡 등록 |
| 현재 작업 범위 **밖** + 심각함 | 제안 + 🔴 긴급 권고 → 원주 판단 |
| `@Reviewer` 리뷰 중 발견 | 리뷰 리포트에 포함 + TODO 제안 |

### 자동 등록 예외 (원주 승인 불필요)

```
5단계 완료 체크리스트 실행 시,
Claude Code가 "다음에 해야 할 것"을 정리하며 등록하는 경우:
  → 이미 원주가 작업 완료를 확인하는 시점이므로 자동 등록 허용
  → 단, 등록 내역을 보여줘야 함:
    "📋 세션 정리 중 TODO 2건 자동 등록:
      • cast_instant 모션 ratio 미세 조정
      • DebugCombatPanel VP 표시 소수점 오류"
```

### CLAUDE.md 추가

```markdown
## TODO 에이전트 규칙
- 에이전트가 TODO를 발견하면 자동 등록하지 않고 "💡 TODO 제안" 후 사용자 승인
- 예외: 5단계 완료 체크리스트 시 자동 등록 허용 (등록 내역 표시 필수)
- 현재 작업 범위 안의 이슈는 즉시 수정 (TODO 아님)
```

## 보강 4: 백로그 카테고리 분류 규칙

### 문제

`/todo backlog` 강등 시 🟢의 어느 서브카테고리(시스템/에셋/폴리시/설계미결정)로 넣을지 규칙 없음.

### 해결: 키워드 기반 자동 분류 + 수동 지정

```
자동 분류 키워드:

시스템:
  구현, 리팩토링, 매니저, 시스템, 함수, 스크립트, 로직, 버그, null

에셋:
  에셋, GLB, FBX, 모델, 텍스처, 에셋 수급, 교체, Quaternius, Kenney

폴리시:
  UI, 시각화, 연출, 애니메이션, 카메라, FX, 이펙트, 디버그, 표시

설계 미결정:
  계수, 확정, 밸런스, 수치, 방식, 미결정, PENDING
```

### 커맨드 확장

```
/todo backlog {키워드}
  → 자동 분류 시도 → 결과 표시

  🟢 강등: HP_BAR_Y 높이 조정 → 폴리시 (자동 분류: "높이", "조정")
  맞으면 Enter, 변경하려면 카테고리 입력 (시스템/에셋/폴리시/설계)

/todo backlog {키워드} --cat 에셋
  → 카테고리 직접 지정
```

### 규칙

- 자동 분류 실패 시(키워드 매칭 없음) → "카테고리를 선택해주세요" 요청
- 최종 판단은 원주 — 자동 분류는 제안일 뿐
- 🟡에서는 카테고리 없음 (flat list). 🟢 진입 시에만 분류

## 보강 5: 검색 커맨드

### 문제

항목 30개+ 시 `/todo list`로 전부 보는 것은 비효율.

### 커맨드

```
/todo search {키워드}

예:
  /todo search 투사체

출력:
  🔍 "투사체" 검색 결과 (3건):

  🟡 #6. ProjectileManager + Projectile 씬 구현
  🟢 [에셋] 투사체 GLB 에셋 교체
  🟢 [폴리시] 투사체 FX Trail 실구현
```

### 규칙

- 모든 우선순위 + ✅ + ❌ 전체 검색
- 항목 텍스트 + 메모 텍스트 모두 검색 대상
- 결과에 우선순위 + 카테고리 표시

### CLAUDE.md 추가

```markdown
- `/todo search {키워드}` — 전체 항목 키워드 검색
```

## 최종 커맨드 전체 목록

| 커맨드 | 동작 |
|--------|------|
| `/todo {내용}` | 🟡에 등록 (중복 감지) |
| `TODO: {내용}` | 🟡에 등록 (자연어) |
| `/todo list` | 🔴 + 🟡 출력 |
| `/todo list -v` | 🔴 + 🟡 출력 (메모 포함) |
| `/todo all` | 전체 출력 |
| `/todo search {키워드}` | 전체 키워드 검색 |
| `/todo done {키워드}` | ✅ 완료 (연쇄 체크) |
| `/todo wontdo {키워드} — {사유}` | ❌ 폐기 (사유 필수) |
| `/todo urgent {키워드}` | 🟡 → 🔴 |
| `/todo backlog {키워드}` | 🟡 → 🟢 (카테고리 자동 분류) |
| `/todo note {키워드} {메모}` | 항목에 컨텍스트 추가 |
| `/todo update {키워드} → {새 내용}` | 내용 수정 + STALE 해제 |
| `/todo review` | Phase 전환 점검 |
| `/todo milestone` | 마일스톤 점검 |
