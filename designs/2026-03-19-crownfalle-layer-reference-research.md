# CrownFalle — 전투 외 레이어 기획 레퍼런스 조사

> 날짜: 2026-03-19 (Banner Saga 추가: 2026-03-19, Wildermyth 추가: 2026-03-19)

> 우선순위: Town/Camp → WorldMap → Roster

## 1. Town / Camp 레퍼런스 비교

### 1-1. Battle Brothers — 마을 시설

Battle Brothers의 정착지는 크기(마을/요새/도시)에 따라 시설과 모집 풀이 달라진다.

**마을 시설 구조:**

| 시설 | 기능 | CrownFalle 참고 |
|------|------|----------------|
| Market | 물품 구매/판매, 3일마다 갱신 | 상점 시스템 |
| Tavern | 용병 모집, 음료로 사기 부스트(과음 패널티), 루머 구매 | 여관: 모집 + 사기 관리 |
| Temple | 부상 치료 시간 단축 (비용 지불) | 치료소 |
| Weaponsmith | 고품질 무기 판매, 수리 | 대장간 |
| Armorsmith | 고품질 방어구/방패 판매, 즉시 수리 | 대장간 |
| Fletcher | 원거리 무기/탄약 판매 | 대장간에 통합 가능 |
| Taxidermist | 트로피 → 아이템 제작 | 제작 시스템 (확장) |
| Barber | 외모 변경 | 불필요 (2D) |
| Docks | 항구 간 즉시 이동 (비용) | 월드맵 빠른 이동 |
| Training Hall | XP 부스트 (비용) | 용병 훈련 |

**핵심 특징:**
- 마을마다 "부속 시설(Attached Location)"이 달라 모집 풀이 변함 (사냥꾼 오두막 → 궁수 모집 가능)
- 마을 상태(전쟁/역병/풍년)가 물가와 모집 풀에 동적 영향
- 시설 업그레이드 없음 — 더 좋은 시설은 더 큰 도시에서 찾아야 함

**CrownFalle 적용 인사이트:**
- 마을마다 차별화된 시설/모집 풀 → 여행의 의미 부여
- 마을 상태 시스템 → 평판(Reputation) 자원과 연결 가능
- 시설 업그레이드 없음은 BB 특유의 노마드 스타일 — CrownFalle는 캠프 업그레이드로 차별화 가능

### 1-2. Darkest Dungeon — 햄릿 (고정 타운)

고정된 1개 허브 타운. 시설을 "유물(Heirloom)"으로 업그레이드하는 메타 진행이 핵심.

**햄릿 시설 구조:**

| 시설 | 기능 | 업그레이드 통화 | CrownFalle 참고 |
|------|------|--------------|----------------|
| Stage Coach | 영웅 모집 (주간 갱신), 로스터 상한 확장 | Deeds + Crests | 여관/모집소 |
| Blacksmith | 무기/방어구 티어 업그레이드 (개별 영웅) | Deeds + Crests | 대장간 |
| Guild | 스킬 해금/업그레이드 (개별 영웅) | Crests + Portraits | 훈련장/스킬 관리 |
| Sanitarium | 기벽(Quirk) 고정/제거, 질병 치료 | Busts + Crests | 치료소 |
| Abbey | 스트레스 해소 (명상/기도/고행) | Busts + Crests | 사기(Morale) 회복 |
| Tavern | 스트레스 해소 (음주/도박/유흥) | Portraits + Crests | 사기(Morale) 회복 |
| Survivalist | 캠핑 스킬 교습 | Crests | 캠프 스킬 |
| Nomad Wagon | 장신구(Trinket) 거래 | — | 악세서리 상점 |

**핵심 특징:**
- 업그레이드 통화가 4종(Deeds/Busts/Portraits/Crests) — 시설 간 경쟁 발생
- 스트레스 해소 시설에 영웅을 "배치"하면 1주간 출격 불가 → 로테이션 강제
- 시설 업그레이드가 게임 진행의 핵심 축 (더 좋은 장비 = 더 높은 던전)
- 기벽/질병이 영속적 상태 → 정신병원이 경제적 싱크

**CrownFalle 적용 인사이트:**
- 사기(Morale) 회복에 시간/비용을 들이면 로테이션이 자연스럽게 발생
- 업그레이드 통화 분리 → 자원 간 선택의 긴장감 (CrownFalle는 4대 자원으로 이미 설계됨)
- 캠프 vs 마을 역할 분리: 캠프 = 간이 회복, 마을 = 본격 치료/업그레이드

### 1-3. Wartales — 이동식 캠프

마을은 거래/모집 허브, 캠프는 이동식 생존 거점.

**캠프 구조물:**

| 구조물 | 기능 | 배치 필요 | CrownFalle 참고 |
|--------|------|----------|----------------|
| Campfire | 식사/급여, 사기 보너스 | 용병 배치 시 행복도+ | 모닥불 |
| Workshop | 수리 도구 자동 생산 | 용병 배치 | 간이 공방 |
| Tent | 수면, Valor 회복 | — | 텐트 |
| Strategy Table | 전투 전략 버프 (배치/정찰 등) | — | 전략 테이블 |
| Cooking Pot | 요리 → 파티 전체 버프 | 요리사 배치 | 요리 시설 |
| Trough | 동물 탑재량/편안함 | 동물 배치 | — (동물 없으면 불필요) |
| Artefact | 고급 버프 (인접 배치 효과) | 위치 중요 | 유물/아티팩트 |

**마을 시설:**

| 시설 | 기능 |
|------|------|
| Inn | 모집, 퀘스트 게시판 |
| Market | 물품 구매/판매 |
| Blacksmith | 장비 제작/수리/강화 |
| Arena (DLC) | 도전 전투 |
| Jail | 포로 매각, 수배도 감소 |

**핵심 특징:**
- 캠프 구조물에 용병을 배치해야 효과 발동 → 전투 인원 vs 캠프 운영의 트레이드오프
- 3번 휴식마다 급여 지급 → 주기적 경제 압박
- 직업(Profession) 시스템: 전투 클래스와 별도로 요리/대장장이/도둑/약초사 등
- 요리 → 버프는 중독성 있는 루프 (좋은 레시피 = 전투 유리)

**CrownFalle 적용 인사이트:**
- 캠프 구조물 "배치" 메카닉 → 용병 할당 결정 (전투 vs 캠프 유틸리티)
- 식량 소모 + 급여 주기 → 경제 압박의 핵심 루프
- 요리 시스템은 2D에서도 구현 가능하고 재미있는 서브 시스템

### 1-4. Banner Saga — 캠프 + 캐러밴

Banner Saga는 고정 타운 없이, 이동하는 캐러밴 자체가 거점 역할을 한다.

**캠프 구조:**

| 시설/기능 | 설명 | CrownFalle 참고 |
|----------|------|----------------|
| 캠프 화면 | 여행 중 아무 때나 캠프 설치 가능 | 이동식 캠프 |
| 영웅 관리 | 캐릭터 레벨업(Renown 소모), 아이템 장착 | 로스터 관리 |
| 동료 대화 | 캠프에서 동료와 대화 → 관계/스토리 진행 | 캠프 내 대화 이벤트 |
| 보급품 구매 | 마을 도착 시 Market에서 식량 구매 | 마을 상점 |
| 휴식 | 캠프에서 1일 휴식 → 사기 회복 (식량 소모) | 캠프 휴식 |

**핵심 특징:**
- 캐러밴 인구가 3종류 (전사/시민/바를) — 각각 식량 소모량, 전투 기여, 채집 능력이 다름
- 사기가 낮으면 부정적 이벤트 확률 증가 (내분, 이탈, 문제 유발)
- 사기 회복은 "숫자 변화가 아니라 대화와 이벤트로 전달" — 경영이 내러티브와 결합
- 마을은 고정 시설이 아니라 스토리 이벤트의 배경으로 기능
- 캠프에서 동료와의 대화가 스토리 진행의 핵심 수단

**CrownFalle 적용 인사이트:**
- 캠프 내 "동료 대화" → 스토리 동료의 개인 퀘스트/관계 진행 장소
- 사기 시스템을 숫자가 아닌 이벤트로 체감시키는 설계 철학
- 캐러밴 인구 관리는 CrownFalle에 그대로 적용하기엔 무겁지만, "비전투 인원 관리" 개념은 참고 가능

### 1-5. Wildermyth — 챕터 간 다운타임 + 오버월드 전략

Wildermyth에는 전통적 타운이 없다. 대신 오버월드 맵에서 지역 확보/방어를 하고, 챕터 사이 "평화 시간"이 타운 역할을 대신한다.

**오버월드 전략 (타운 대체):**

| 기능 | 설명 | CrownFalle 참고 |
|------|------|----------------|
| 지역 확보 | 탐험 → 자원 제공 지역 확보 | 월드맵 지역 점령? |
| 방어 구축 | 확보 지역에 방어 시설 건설 | 캠프 구조물의 확장 버전 |
| 시간 관리 | 탐험/건설/전투 사이 시간 배분 | 여행 중 시간 경과 |
| 챕터 간 평화 시간 | 영웅이 연구/상업/은퇴 등 개인 활동 | 캠프/마을 다운타임 이벤트 |

**핵심 특징:**
- "평화 시간"에 영웅이 자율적으로 활동 (도끼 공기역학 연구, 상점 운영, 농부 은퇴 등) → 캐릭터에 생동감 부여
- 타운 시설이 아닌 "시간의 경과" 자체가 캐릭터 발전의 수단
- 적 세력이 시간에 비례하여 강해짐 → 다운타임의 기회비용
- 6명 팀으로 개발한 인디 게임 — 솔로 개발에 참고할 스코프

**CrownFalle 적용 인사이트:**
- 캠프 휴식 시 "다운타임 이벤트" 도입 가능 — 용병들이 자율 활동하며 관계/성격 변화
- 시간 경과에 따른 적 강화 → 여행/휴식의 기회비용 발생, 플레이어에게 긴장감 부여
- 타운 시설 구현 없이도 "시간 + 이벤트"로 경영 깊이를 줄 수 있다는 증거

### 1-6. Dust and Salt — 타운 개념 없음

Dust and Salt에는 독립된 타운 레이어가 없다. 마을 방문은 텍스트 이벤트로 처리되며, 시설이 아닌 "선택지"로 상호작용한다.

**CrownFalle 적용 인사이트:**
- 마을의 시각적 표현이 일러스트 + 텍스트만으로도 충분히 작동할 수 있다는 증거
- 구현 복잡도를 낮추면서 분위기를 살리는 방법

## 2. World Map 레퍼런스 비교

### 2-1. Dust and Salt — 포인트 클릭 (확정 방식)

- 일러스트 맵 위 클릭 가능한 위치
- 위치 클릭 → 즉시 이동 → 텍스트 이벤트 시작
- 이동 비용/소모 없음 (CrownFalle는 식량+피로도 추가)
- 지역: Northblaze, Oakleaf, Outremer, Discord, Murk 등 5~6개

### 2-2. Battle Brothers — 노드 기반 자유 이동

- 도로 기반 이동, 비포장 지역도 가능 (느림)
- 적 무리가 맵 위 실시간 이동 → 접촉 시 전투
- 계약(Contract) 시스템: 마을 게시판에서 수주 → 목표 지점 이동 → 완수
- 마을 간 거리 = 이동 시간 = 식량 소모
- 밤/낮 사이클, 날씨 영향

**CrownFalle 적용 인사이트:**
- 포인트 클릭 맵에서도 "거리" 개념 도입 가능 (위치 간 거리 → 식량/피로도 소모량 차등)
- 계약/의뢰 시스템 → 월드맵 이동의 목적 부여

### 2-3. Banner Saga — 캐러밴 여행 경로

- 일러스트 맵 위 캐러밴이 고정 경로를 따라 이동 (자유 이동 아님)
- 이동 중 식량 소모 (인구 비례), 사기 감소 (장거리 행군)
- 경로 중간에 이벤트 마커 배치 → 텍스트 선택지 팝업
- 캐러밴을 멈추고 캠프 설치 가능 (휴식 = 1일 소모 = 식량 감소)
- 특정 분기점에서 경로 선택 (A길 vs B길 → 다른 이벤트/만남)

**핵심 자원:**
- Supplies (식량): 인구에 비례하여 소모, 0이 되면 사기 급락 + 아사 발생
- Morale: 여행 길이, 선택 결과, 전투 결과에 따라 변동 → 이벤트 풀 영향
- Population (전사/시민): 전투 시 전사 수가 "전쟁 이벤트" 결과에 영향

**CrownFalle 적용 인사이트:**
- 포인트 클릭이지만 "경로" 개념을 도입하면 Banner Saga식 여행 긴장감 구현 가능
- 위치 A→B 이동 시 중간에 1~2개 이벤트 발생 → 단순 클릭이 아닌 "여정"으로 체감
- 식량 소모를 인원 수에 비례시키면 로스터 크기와 여행 비용의 트레이드오프 발생

### 2-4. Wildermyth — 오버월드 전략 맵

- 2D 일러스트 오버월드 맵에서 지역을 탐험/확보/방어
- 영웅을 보내 지역 탐험 → 발견된 위협에 대응 (전투 진입)
- 지역 확보 시 자원 제공 + 방어 시설 건설 가능
- 적 세력이 시간에 따라 영토 확장 → "탐험 vs 방어 vs 공격" 시간 배분 전략
- 챕터가 진행될수록 맵 확장

**CrownFalle 적용 인사이트:**
- 포인트 클릭 맵의 각 위치에 "확보/미확보" 상태 부여 → 확보 지역은 안전하게 이동, 미확보 지역은 이벤트/전투 발생
- 시간 경과에 따라 적 세력이 확장되면 플레이어에게 "어디를 먼저 갈 것인가" 전략적 선택 부여

### 2-5. Darkest Dungeon — 월드맵 없음 (던전 선택)

- 월드맵 대신 "던전 선택 화면" — 4개 지역 중 선택
- 지역별 난이도 3단계 (Apprentice/Veteran/Champion)
- 던전 진입 전 보급품 구매

**CrownFalle 적용 인사이트:**
- 출격 전 보급품 구매 → CrownFalle의 "여행 전 식량 준비"와 동일 구조

### 2-7. CrownFalle WorldMap 기획에 필요한 결정 사항

## 3. Roster 레퍼런스 비교

### 3-1. Battle Brothers — 용병 모집/관리

- 모집 시 스탯 숨김 (배경+비용으로 추측, 고용 후 확인)
- 배경(Background) 시스템: 농부/도적/기사/사냥꾼 등 → 스탯 경향성 결정
- 용병당 일일 급여 (배경에 따라 차등)
- 장비 = 전투에서 루팅 or 상점 구매
- 영구 사망 + 부상(Permanent Injury)
- 로스터 12명 (초기) → 최대 20명

**CrownFalle 적용 인사이트:**
- 배경 시스템 → CrownFalle의 "무기 기반 정체성"과 결합 가능 (배경이 초기 스탯 경향+시작 무기 결정)
- 모집 시 정보 제한 → 고용의 리스크/리워드
- 일일 급여 → Gold 자원 압박의 핵심 드라이버

### 3-2. Darkest Dungeon — 영웅 로스터

- 매주 Stage Coach에서 신규 영웅 도착 (무료 모집, 기본 레벨)
- 15+ 고정 클래스, 각각 고유 스킬 세트
- 기벽(Quirk) 시스템: 긍정 5개 + 부정 5개 + 질병 3개 (랜덤 획득)
- 스트레스 → 고난(Affliction) or 미덕(Virtue)
- 영구 사망 (Death's Door → Deathblow)
- 로스터 상한 업그레이드 (Stage Coach)

**CrownFalle 적용 인사이트:**
- 기벽(Quirk) → CrownFalle의 성향(Trait/Disposition) 시스템과 매핑 가능
- 스트레스/고난 → 사기(Morale) 시스템으로 흡수 가능 (전투 중 사기 저하 → 이탈/디버프)
- 무료 모집 + 소모적 사용 vs BB의 유료 모집 + 투자형 사용 → CrownFalle는 유료 모집 (BB식)

### 3-3. Wartales — 용병 성장

- 마을 여관에서 모집 (유료)
- 7종 전투 클래스 + 전문화 트리
- 전투 클래스와 별도로 직업(Profession) 1개 (요리/대장장이/약초사 등)
- 장비 제작/강화 시스템 (재료 기반)
- 로스터 12~20명+
- 포로 시스템 (적 생포 → 용병 전환 가능)

**CrownFalle 적용 인사이트:**
- 전투 클래스 + 생활 직업 이중 구조 → CrownFalle의 무기 기반 정체성 + 캠프 역할?
- 포로 전환 → 적 용병 편입 (선택적 확장)
- 장비 제작 → 캠프/마을 시설과 연결

### 3-4. Wildermyth — 절차적 캐릭터 성장 + 레거시

- 3개 클래스 (Warrior/Hunter/Mystic) — 레벨업 시 대규모 스킬 뱅크에서 선택
- 영웅이 노화 (젊음 → 전성기 → 노년) — 스탯에 영향, 은퇴/사망 시 레거시로 보존
- 이벤트를 통한 변형 (Transformation): 늑대 혈통 발견 → 외모+능력 변화, 불 조작 능력 획득 등
- 관계 시스템: 사랑/라이벌 → 공유 버프 + 추가 이벤트
- 자녀 시스템: 연인 관계 영웅의 자녀가 성장 후 파티 합류
- Death 선택지: HP 0 시 "불구로 후퇴 (1회 한정)" vs "영광의 죽음 (파티 버프)"
- 레거시 시스템: 은퇴/사망 영웅이 "전설"로 저장 → 다음 캠페인에서 재등장 가능

**핵심 특징:**
- "사건이 캐릭터를 물리적으로 변형한다" — 단순 스탯 버프가 아닌 외모/능력의 근본적 변화
- 수제 이벤트 + 절차적 조합 = 매 플레이스루마다 다른 캐릭터 아크
- 6명 팀으로 95% 압도적 긍정 리뷰 달성 — 솔로/소규모 개발의 벤치마크

**CrownFalle 적용 인사이트:**
- 성향(Trait/Disposition) 시스템을 "이벤트 기반 변형"으로 확장 — 레벨업 마일스톤이 아니라 스토리 이벤트에서 성향 획득
- Death 선택지 (불구 vs 영광의 죽음) → CrownFalle의 영구 사망에 "마지막 순간의 선택" 추가
- 노화 시스템은 프로토타입 스코프 밖이지만, 장기 비전으로 고려 가능
- 관계 시스템 → Banner Saga 동료 대화와 결합하여 캠프에서 관계가 발전하는 구조

### 3-5. Banner Saga — 스토리 기반 동료

- 동료는 스토리 이벤트/선택지를 통해 합류 (랜덤 모집 없음)
- 각 동료는 고유 이름, 배경 스토리, 고유 스킬 보유
- 선택에 따라 동료가 합류/이탈/사망 — 플레이어 행동의 결과
- 캠프에서 동료와 대화 → 관계 심화, 스토리 분기에 영향
- 동료 성장: Renown(명성) 포인트로 레벨업 (전투 보상 + 이벤트 보상)
- 아이템 장착 (액세서리 슬롯)으로 능력 커스터마이징
- 트릴로지 전체에 걸쳐 동료 생존 여부가 이어짐

**핵심 특징:**
- 모집이 "선택"이 아니라 "결과" — 특정 이벤트에서 올바른 선택을 해야 합류
- 동료 사망도 "선택의 결과" — 잘못된 판단 시 영구 손실
- 동료마다 전투 역할이 뚜렷 (탱커/궁수/서포터 등)
- 스토리 동료 수는 30명+ (전체 트릴로지)이지만, 한 시점에 사용 가능한 수는 제한적

**CrownFalle 적용 인사이트:**
- 하이브리드 모집 구조 가능: 스토리 동료 (Banner Saga식) + 일반 용병 (BB식)
- 스토리 동료는 고유 스킬/대사/개인 퀘스트 → 감정 투자 유도
- 일반 용병은 소모적 운용 → 전술적 깊이 + 경제 압박
- 스토리 동료의 사망 = 내러티브 임팩트, 일반 용병의 사망 = 경영 손실

### 3-7. CrownFalle Roster 기획에 필요한 결정 사항

| 결정 항목 | 후보 | 레퍼런스 |
|----------|------|---------|
| 모집 구조 | 스토리 동료만 vs 일반 용병만 vs **하이브리드** | Banner Saga (스토리) + BB (용병) |
| 모집 비용 구조 | 유료 (배경에 따라 차등) vs 무료 vs 혼합 | BB (유료) vs DD (무료) vs Banner Saga (무료/이벤트) |
| 모집 시 정보 공개 | 전체 공개 vs 부분 숨김 vs 전부 숨김 | BB (숨김) vs Wartales (공개) |
| 배경/출신 시스템 | 있음 (스탯 경향성) vs 없음 (순수 스탯) | BB (있음) |
| 레벨업 방식 | 자동 성장 vs 포인트 배분 vs 선택형 | DD (스킬 선택) vs BB (스탯 선택) vs Banner Saga (Renown) |

| 스토리 동료 수 | 5~10명 (소수 정예) vs 15명+ (대규모) | Banner Saga (30+) vs Lost Eidolons (중간) |
| 캐릭터 변형 (Transformation) | 이벤트로 외모/능력 변화 vs 없음 | Wildermyth (핵심) |
| 레거시/은퇴 시스템 | 있음 (전설로 보존) vs 없음 | Wildermyth (있음, 장기 비전) |
