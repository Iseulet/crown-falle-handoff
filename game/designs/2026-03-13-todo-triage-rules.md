## 1. 점검이 발생하는 3가지 시점

```
┌─────────────────────────────────────────────────┐
│                                                  │
│   Phase 내 개발 진행 중                           │
│   ┌────┐  ┌────┐  ┌────┐  ┌────┐               │
│   │구현│→│테스트│→│구현│→│테스트│→ ...           │
│   └────┘  └────┘  └────┘  └────┘               │
│       ↑               ↑                         │
│    (B) 임계값        (B) 임계값                   │
│     도달 체크         도달 체크                    │
│                                                  │
│   ───────────────────────────────── Phase 완료 ──│
│                                          ↑       │
│                                    (A) 필수 점검  │
│                                                  │
│   ─────── 다음 Phase 시작 ───────────────────────│
│       ↑                                          │
│    (C) 마일스톤                                   │
│     연동 점검                                     │
└─────────────────────────────────────────────────┘
```

## 2. (A) Phase 전환 점검 — 필수

Phase가 끝나고 다음 Phase를 시작하기 전에 **반드시** 실행.

### 트리거

```
Phase 완료 선언 시 (커밋 + "Phase X 완료" 확인)
  → Claude Code가 자동 제안:
    "📋 Phase 완료. TODO 점검을 실행할까요? (/todo review)"
```

### 점검 내용

```
/todo review 실행 시:

1. 현황 요약
   🔴 긴급: N개  🟡 다음: N개  🟢 백로그: N개  ✅ 완료: N개

2. 🟡 다음 → 다음 Phase 관련성 분류
   - 다음 Phase에 필수 → 🟡 유지
   - 다음 Phase와 무관 → 🟢 백로그로 강등 제안
   - 이미 해결됨 → ✅ 완료 제안

3. 🟢 백로그 → 다음 Phase에서 처리 가능한 것 승격 제안
   - 다음 Phase 설계와 겹치는 항목 → 🟡 승격 제안

4. 완료 아카이브
   - 지난 Phase의 ✅ 항목 정리
```

### 출력 형식

```
📋 Phase B 완료 — TODO 점검 결과

현황: 🔴 0 / 🟡 8 / 🟢 16 / ✅ 4

🟡 → 🟢 강등 제안 (다음 Phase 무관):
  • HP_BAR_Y 높이 조정 — 비주얼 폴리시, 급하지 않음
  • Archer idle 모션 속도 — 에셋 튜닝

🟢 → 🟡 승격 제안 (다음 Phase 관련):
  • AoE 마법 설계 — Phase C가 스킬 확장이면 필요

유지:
  • 나머지 6개 🟡 항목은 그대로 유지

원주의 결정이 필요합니다. 제안대로 처리할까요?
```

원주가 승인하면 일괄 이동. 개별 항목 수정도 가능.

## 3. (B) 임계값 기반 중간 체크 — 자동 알림

Phase 진행 중에 TODO 항목이 특정 임계값을 넘으면 Claude Code가 알린다.

### 임계값

| 우선순위 | 임계값 | 알림 |
|---------|--------|------|
| 🔴 긴급 | **1개 이상** | 즉시 알림 — 현재 작업 차단 가능성 경고 |
| 🟡 다음 | **15개 이상** | 세션 시작 시 알림 — 정리 세션 제안 |
| 🟡 다음 | **20개 이상** | 작업 중 알림 — 강력 정리 권고 |

### 체크 시점

```
임계값 체크가 실행되는 순간:
  1. CLI 세션 시작 시 (TODO.md 읽을 때)
  2. /todo 등록 시 (항목 추가 직후)
  3. Phase 완료 시 (A와 겹침)
```

### 알림 예시

```
⚠️ TODO 임계값 도달

🟡 다음: 17개 (임계값 15)
최근 5일간 +9개 등록, -2개 완료 (순증가 +7)

정리 세션을 권고합니다:
  • /todo review — 전체 점검 + 분류 제안
  • /todo list — 현재 목록 확인 후 직접 정리

지금 정리할까요, 아니면 현재 작업 계속할까요?
```

원주가 "계속"하면 다음 세션까지 재알림 없음.

### 🔴 긴급 알림 (즉시)

```
🚨 긴급 TODO 감지

🔴 긴급: 2개
  1. 전투 씬 크래시 — null 참조 (CombatUnit.receive_hit)
  2. 교전 해제 시 무한루프

현재 작업보다 우선 처리를 권고합니다.
확인 후 진행 방향을 알려주세요.
```

## 4. (C) 마일스톤 연동 점검

### 현재 마일스톤 구조

```
Phase 1~5: 기초 프로토타입 (완료)
Phase A: 3D 전환 + 애니메이션 (완료)
Phase B: 투사체 + 원거리 (완료)
Phase C: (미정 — 스킬 확장? 월드맵 리워크? UI 폴리시?)
```

### 마일스톤 전환 시 점검

Phase가 아닌 **마일스톤 단위**(여러 Phase를 묶은 큰 단위)의 전환점에서는 더 넓은 시야로 점검한다.

```
/todo milestone 실행 시:

1. 전체 백로그 규모
   총 N개 (🔴 a / 🟡 b / 🟢 c)

2. 카테고리별 분포
   시스템: N개 / 에셋: N개 / 폴리시: N개 / 설계 미결정: N개

3. 오래된 항목 하이라이트
   7일 이상 🟡에 머문 항목 → 처리 또는 🟢 강등 제안

4. 다음 마일스톤 방향 제안
   "🟡에 에셋 항목이 5개 집중 — 에셋 수급 Phase를 별도로?"
   "설계 미결정이 7개 — 밸런스 결정 세션 필요?"
```

이건 원주가 "다음에 뭘 할지" 방향을 잡을 때 참고 자료가 된다.

## 5. 정리 세션 운영 방식

### 경량 정리 (사이사이)

🟡 15개 미만이거나, 원주가 "지금은 됐어"인 경우.

```
세션 시작 시 /todo list 한번 훑기
  → 이미 해결된 것 /todo done 처리
  → 급해진 것 /todo urgent 승격
  → 3분 이내 완료
```

### 전용 정리 세션

🟡 15개 이상이거나, Phase 전환 시.

```
1. /todo review 실행 (Claude Code 자동 분석)
2. 제안 검토 (승격/강등/완료/유지)
3. 원주 승인 → 일괄 처리
4. 다음 Phase 🟡 목록 최종 확인
5. 10~15분 소요
```

### 판단 기준 요약

```
🟡 < 10개:  정리 불필요. 개발 계속.
🟡 10~14개: 세션 시작 시 빠르게 훑기 (3분)
🟡 15~19개: 정리 세션 제안. 원주 판단.
🟡 20개+:   강력 권고. 정리 안 하면 다음 세션에도 재알림.
🔴 1개+:    즉시 알림. 현재 작업 차단 가능성.
```

## 6. TODO.md 헤더에 현황 카운터 추가

TODO.md 최상단에 항상 현재 카운트를 기록한다.
등록/완료/이동 시 Claude Code가 자동 갱신.

```markdown
# CrownFalle — TODO
> 최종 갱신: 2026-03-13
> 현황: 🔴 0 / 🟡 8 / 🟢 16 / ✅ 4
```

`/todo list` 출력 시에도 이 카운터가 맨 위에 표시된다.

## 7. CLAUDE.md 추가 규칙

기존 TODO 관리 섹션에 추가:

```markdown
## TODO 점검 타이밍

- Phase 완료 시: `/todo review` 자동 제안 (필수)
- 세션 시작 시: 🔴 1개+ → 즉시 알림, 🟡 15개+ → 정리 제안
- /todo 등록 시: 등록 직후 임계값 체크
- `/todo review` — Phase 전환 점검 (분류 제안 + 일괄 처리)
- `/todo milestone` — 마일스톤 전환 점검 (전체 분포 + 방향 제안)
```

## 8. 전체 TODO 커맨드 요약

| 커맨드 | 동작 | 시점 |
|--------|------|------|
| `/todo {내용}` | 🟡에 등록 | 언제든 |
| `TODO: {내용}` | 🟡에 등록 (자연어) | 언제든 |
| `/todo list` | 🔴 + 🟡 출력 | 언제든 |
| `/todo all` | 전체 출력 | 언제든 |
| `/todo done {키워드}` | ✅로 이동 | 언제든 |
| `/todo urgent {키워드}` | 🟡 → 🔴 | 언제든 |
| `/todo backlog {키워드}` | 🟡 → 🟢 | 언제든 |
| `/todo review` | Phase 전환 점검 | Phase 완료 시 (자동 제안) |
| `/todo milestone` | 마일스톤 점검 | 큰 방향 전환 시 |
