## 1. 설계 철학

### 각 게임에서 가져오는 것

```
워테일즈에서:
  ✅ 자유 선택 턴제 (아군 유닛 행동 순서 자유 결정)
  ✅ 교전(Engagement) 시스템 (이동 봉쇄 + 기회 공격)
  ✅ 부상(Injury) 시스템 (전투 후 영구 페널티)
  ✅ 자원 관리 긴장감 (STA/MP 이원화 + VP 삼원화)

DOS2에서:
  ✅ 방어 게이지 → CC 면역 게이트 구조
  ✅ 유틸리티 스킬의 전략적 가치 (밀치기, 끌어오기, 교전 조작)
  ✅ 원소 속성 상성 (물→화염 반감, 물→전기 증폭)
  ✅ 단계적 CC 전환 (물+냉기→빙결)
  ✅ 지형 효과 레이어 (후순위로 도입)

가져가지 않는 것:
  ❌ DOS2의 AP 시스템 → STA/MP 이원화 유지
  ❌ DOS2의 Initiative 고정 턴제 → 자유 선택 턴제 유지
  ❌ DOS2의 소스 포인트 → VP(사기) 변형으로 대체
  ❌ DOS2의 고저차 시스템 → 프로토타입 범위 초과
```

## 3. [A] 자원 시스템 — STA / MP / VP 삼원화

### STA & MP (기존 확정 유지)

| 자원 | 공식 | 용도 |
|------|------|------|
| STA | 10 + CON × 2 | 이동, 물리 공격, 물리 스킬, 교전 유지 |
| MP | WIL × 3 | 마법 스킬 |

### VP — 사기 (신규, 파티 공유 자원)

| 항목 | 설계 |
|------|------|
| **정체성** | "사기(Morale)" — 용병단의 전투 의지 |
| **공유 자원** | 파티 전체가 공유하는 단일 풀 (개인 자원 아님) |
| **초기값** | 전투 시작 시 파티 인원 × 1 (4인 = 4VP) |
| **수급** | 적 처치 +1, 치명타 +1, 교전 성립 시 특정 직업 +1 |
| **소모** | 궁극기급 액티브 스킬, 조건부 이탈, 전투 중 부상 회복 |
| **전투 후** | 잔여 VP 소멸 (캠프 휴식으로 초기값 복구) |

**설계 의도:**
- 인원×1로 빡빡하게 시작 → 모든 VP 사용이 무게 있는 결정
- 적 처치/치명타로 수급 → 공격적 플레이에 보상
- 궁극기 비용 2~3VP → 초기 풀로는 1회 사용이 한계, 수급해야 재사용 가능

## 4. [B] 방어 시스템 — ARM/RES 게이지형 전환

### 기존 %형 폐기, 게이지형으로 교체 ⚠️ 스탯 공식 변경

기존 ARM(%) = CON × 0.3, RES(%) = WIL × 0.3 → **폐기**

| 방어 | 신규 공식 | 역할 |
|------|----------|------|
| ARM 게이지 | CON × `arm_per_con` | 물리 공격이 먼저 ARM을 깎음. ARM > 0이면 물리 CC 면역. |
| RES 게이지 | WIL × `res_per_wil` | 마법 공격이 먼저 RES를 깎음. RES > 0이면 마법 CC 면역. |

### 데미지 흐름 (변경)

```
물리 공격:
  1. ARM 게이지 > 0?
     YES → 데미지를 ARM에서 차감. 잔여 ARM = max(0, ARM - 데미지)
            초과 데미지는 HP로 전이하지 않음 (ARM이 흡수)
            물리 CC 면역 유지 (ARM이 1이라도 남아있으면)
     NO  → 데미지가 HP에 직접 적용. 물리 CC 적용 가능.

마법 공격:
  1. RES 게이지 > 0?
     YES → 데미지를 RES에서 차감. 잔여 RES = max(0, RES - 데미지)
            초과 데미지는 HP로 전이하지 않음
            마법 CC 면역 유지
     NO  → 데미지가 HP에 직접 적용. 마법 CC 적용 가능.
```

### combat_config.jsonc 변경

```jsonc
// 기존 stat_derivation 에서 제거:
//   "con_arm_coefficient": 0.3  ← 폐기
//   "wil_res_coefficient": 0.3  ← 폐기

// 신규 추가:
"armor": {
  // ARM 게이지 = CON × arm_per_con
  "arm_per_con": 3,
  // RES 게이지 = WIL × res_per_wil
  "res_per_wil": 3
}

// 기존 damage 섹션에서 제거:
//   "arm_max": 80  ← 폐기 (%형이 아니므로 불필요)
//   "res_max": 80  ← 폐기
```

### 직업별 초기 방어 게이지

| 직업 | CON | ARM 게이지 (×3) | WIL | RES 게이지 (×3) |
|------|-----|----------------|-----|----------------|
| Fighter | 8 | **24** | 2 | **6** |
| Archer | 5 | **15** | 3 | **9** |
| Rogue | 4 | **12** | 2 | **6** |
| Mage | 4 | **12** | 8 | **24** |

**설계 의도:**
- Fighter = ARM이 두껍고 RES가 얇음 → 마법 CC에 취약
- Mage = RES가 두껍고 ARM이 얇음 → 물리 CC에 취약
- 물리 파티는 ARM 집중 파괴 후 물리 CC, 마법 파티는 RES 집중 파괴 후 마법 CC
- DOS2의 "점사 전략"이 자연스럽게 유도됨

### 전투 후 회복

- ARM/RES 게이지: 전투 후 전량 회복 (HP/STA와 동일)

## 5. [C] 교전 시스템 — 기존 확장 + 조건부 이탈

### 기존 확정 (유지)

| 항목 | 현재 설계 |
|------|----------|
| 교전 성립 | 자유로운 적에게 근접 공격 시 |
| 효과 | 이동 봉쇄, 공격 대상 제한 |
| 해제 | 상대 사망 시 자동 해제 |
| STA 비용 | 교전 유지 턴당 5 (회복과 상쇄) |

### 신규: 조건부 이탈 + 기회 공격

| 이탈 방법 | 비용 | 기회 공격 | 출처 |
|-----------|------|----------|------|
| **강제 이탈** | 없음 | ✅ 기회 공격 발생 (상대 일반 공격 1회) | 워테일즈 |
| **VP 이탈** | VP 1 소모 | ❌ 기회 공격 없음 | 워테일즈 VP |
| **스킬 이탈** | 특정 스킬 사용 (STA/MP 소모) | ❌ 기회 공격 없음 | DOS2 유틸리티 |
| **아군 지원 이탈** | 아군이 도발/밀치기 사용 | ❌ 기회 공격 없음 | DOS2 + 워테일즈 |

### 교전 유도/파괴 스킬 (신규)

| 스킬 유형 | 효과 | 예시 |
|-----------|------|------|
| **도발(Taunt)** | 대상을 자신과 강제 교전 | Fighter 전용 |
| **밀치기(Push)** | 대상을 1~2칸 밀침 → 교전 해제 | 범용 물리 |
| **끌어오기(Pull)** | 대상을 자신 인접으로 끌어옴 → 교전 유도 | 마법/물리 |
| **연막(Smoke)** | 범위 내 교전 해제 + 2턴 은폐 | Rogue 전용 |

## 6. [D] 스킬 시스템

### 6.1 스킬 분류 체계

| 분류 | 자원 | 특징 | 상태 |
|------|------|------|------|
| **패시브** | 없음 (자동 발동) | trigger + condition + effects | ✅ 설계 완료 |
| **액티브 (물리)** | STA | 직접 공격/방어 행동 | 🔄 신규 설계 필요 |
| **액티브 (마법)** | MP | 원소 속성 공격/치유/버프 | 🔄 신규 설계 필요 |
| **궁극기** | VP | 판도를 뒤집는 결정적 기술 | 🔄 신규 설계 필요 |
| **유틸리티** | STA 또는 MP | 위치 이동, 교전 조작 | 🔄 신규 설계 필요 |

### 6.2 액티브 스킬 구조 (제안)

```json
{
  "id": "skill_shield_bash",
  "name": "방패 강타",
  "class": "fighter",
  "type": "active_physical",
  "resource": "sta",
  "cost": 5,
  "range": 1,
  "target": "enemy",
  "damage": {
    "base_stat": "str",
    "coefficient": 0.8
  },
  "element": null,
  "effects": [
    {
      "type": "cc",
      "cc_type": "stun",
      "gate": "arm",
      "duration": 1,
      "chance": 100
    },
    {
      "type": "push",
      "distance": 1,
      "chance": 100
    }
  ],
  "cooldown": 2,
  "animation": "attack_shield_bash"
}
```

**핵심 필드:**
- `"element"`: 스킬의 원소 속성. `null` / `"fire"` / `"lightning"` / `"water"` 등
- `"gate"`: CC 적용 조건. `"arm"` = ARM 게이지 0일 때만 / `"res"` = RES 게이지 0일 때만 / `null` = 게이트 없이 즉시 적용

### 6.3 직업별 스킬 방향성

| 직업 | 역할 | 스킬 방향 | 핵심 자원 |
|------|------|----------|----------|
| **Fighter** | 전열 유지, 교전 고정 | 도발, 방패 기술, ARM 게이지 파괴, 물리 CC | STA |
| **Archer** | 원거리 화력, 견제 | 사거리 공격, 함정 설치, 속성 화살(장비 의존) | STA |
| **Rogue** | 암살, 교전 조작 | 백어택, 연막, 독(장비 의존), 어둠 활용 | STA |
| **Mage** | 원소 마법, CC | 화/전/물 속성 스킬, RES 게이지 파괴, 단계적 CC | MP |

### 6.4 궁극기 (VP 소모)

직업당 1~2개. VP가 빡빡(인원×1)하므로 매우 결정적인 순간에만 사용.

| 직업 | 궁극기 (예시) | VP 비용 | 효과 |
|------|-------------|---------|------|
| Fighter | 전선 돌파 | 2 | 직선 3칸 돌진, 경로 위 적 모두 밀치기 + 데미지 |
| Archer | 화살비 | 3 | 범위 내 모든 적에게 공격 + 이동력 감소 |
| Rogue | 그림자 일격 | 2 | 이동 제한 없이 아무 적에게 순간이동 + 배후 공격 |
| Mage | 원소 폭풍 | 3 | 범위 내 원소 속성 광역 공격 + 상태이상 |

## 7. [E] 원소 시스템 — 6종 / 3레이어

### 7.1 원소 귀속 구조

```
[장비/아이템 레이어] — 클래스 무관, 누구나 접근
  독(Poison): 무기 도포, 독병 투척
  ※ 화/전/물 속성 무기·투척물도 여기 (다른 클래스의 부분 접근 경로)

[스킬 레이어] — 스킬 개별 속성 부여
  화염(Fire) / 전기(Lightning) / 물(Water → 빙결 전환)
  Mage가 이 속성 스킬을 주로 보유. INT 스케일링.
  적성 개념 없음 — Mage는 모든 원소 스킬을 자유롭게 조합.

[지형 레이어] — 전장 환경 종속
  어둠(Dark): 시야 제한 지역, 경계를 넘는 공격에 적중률 페널티
```

**핵심 원칙:** 원소는 캐릭터가 아니라 **스킬과 아이템에 붙는 것**.
어떤 클래스든 해당 속성 스킬이나 아이템을 보유하면 사용 가능.

### 7.2 원소별 상세

| 원소 | 레이어 | 주요 효과 | 상태이상 | 비고 |
|------|--------|----------|---------|------|
| **독 (Poison)** | 장비 | 매 턴 독 데미지 (DoT) | 중독 | 클래스 무관, 무기 도포/투척 |
| **화염 (Fire)** | 스킬 | 화염 데미지 + 지속 화상 | 연소 (DoT) | 물 상태와 반감 관계 |
| **전기 (Lightning)** | 스킬 | 전기 데미지 + 이동력 감소 | 감전 (디버프) | 물 상태와 증폭 관계 |
| **물 (Water)** | 스킬 | 물 속성 부여 + 상태 전환 허브 | 젖음 (Wet) | 화염 반감 / 전기 증폭 / +냉기→빙결 |
| **빙결 (Ice)** | 물의 상태 전환 | 행동 불능 CC | 빙결 (Frozen) | 물 상태 + 냉기 스킬로 전환 |
| **어둠 (Dark)** | 지형 | 시야 제한 → 적중률 페널티 | — (데미지 원소 아님) | 전장 제어 전용 |

### 7.3 속성 상성

```
물(Wet) → 화염(Fire): 반감
  물 상태의 유닛이 화염 공격을 받으면 화염 데미지 감소
  물 상태가 화염 공격을 흡수하여 해제됨

물(Wet) → 전기(Lightning): 증폭
  물 상태의 유닛이 전기 공격을 받으면 전기 데미지 증가
  감전 상태이상 발동 확률 또는 효과 강화

물(Wet) + 냉기 스킬 → 빙결(Frozen): 상태 전환
  젖음 상태의 적에게 냉기 속성 스킬 사용 시 빙결(CC)로 전환
  빙결은 RES 게이트 필요 (RES = 0일 때만 CC 발동)
```

### 7.4 어둠(Dark) 상세 규칙

```
어둠 지역은 전장 위의 영역(Area)으로 존재.
어둠 스킬이나 환경으로 생성.

적중률 페널티 규칙:
  어둠 안 → 어둠 밖 공격: 적중률 감소
  어둠 밖 → 어둠 안 공격: 적중률 감소
  어둠 안 → 어둠 안 공격: 페널티 없음 (둘 다 안에 있으므로)

전술적 함의:
  Rogue가 어둠 안에서 근접 공격 → 페널티 없음 + 은신 시너지
  원거리 적이 어둠 안의 아군을 공격 → 적중률 감소로 아군 보호
  어둠 경계를 만드는 것 자체가 전장을 분할하는 전술적 행위
```

## 8. [F] 상태이상 시스템

### CC (군중 제어) — 방어 게이지 게이트

| CC | 게이트 | 효과 | 지속 |
|----|--------|------|------|
| **기절(Stun)** | ARM = 0 | 행동 불능, 다음 턴 스킵 | 1턴 |
| **넘어짐(Knockdown)** | ARM = 0 | 행동 불능, 기상에 STA 소모 | 1턴 |
| **빙결(Frozen)** | RES = 0 | 행동 불능, 물리 피해 시 추가 데미지 | 1턴 |
| **매혹(Charm)** | RES = 0 | 다음 턴 아군 공격 | 1턴 |
| **공포(Fear)** | RES = 0 | 무작위 방향 이동 후 턴 종료 | 1턴 |

### DoT / 디버프 — 게이트 없음 (방어 게이지 무시)

| 상태 | 원소 | 효과 | 지속 |
|------|------|------|------|
| **중독(Poisoned)** | 독 | 매 턴 독 데미지 | 3턴 |
| **연소(Burning)** | 화염 | 매 턴 화염 데미지 | 2턴 |
| **감전(Shocked)** | 전기 | 이동력 감소 | 2턴 |
| **젖음(Wet)** | 물 | 화염 반감, 전기 증폭, +냉기→빙결 전환 가능 | 2턴 |
| **냉기(Chilled)** | 물(냉기) | 이동력 감소, 재적용 시 빙결 전환 | 2턴 |
| **출혈(Bleeding)** | 물리 | 매 턴 물리 데미지 | 3턴 |

### 단계적 CC 전환

```
젖음(Wet) + 냉기 스킬 → 빙결(Frozen)   ※ RES 게이트
냉기(Chilled) + 냉기 스킬 → 빙결(Frozen)  ※ RES 게이트
젖음(Wet) + 전기 스킬 → 감전 효과 증폭    ※ 게이트 없음 (디버프)

1회 적용 = 디버프 (게이트 없이 즉시)
누적/조합 = CC 전환 (게이트 필요)
```

| 데이터 기반 설계 | ✅ 유지. 모든 신규 수치도 JSON 외부화. |
| 애니메이션 시스템 (Doc 1~6) | ✅ 유지. 스킬 애니메이션은 기존 모션 시스템 활용. |
| 카메라 시스템 | ✅ 유지. 궁극기/CC 발동 시 카메라 연출 연계. |
