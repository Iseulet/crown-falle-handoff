# 에이전트 파이프라인 v3 — 투입 순서 및 역할

> 이전 버전(v2.0)에서 변경된 사항:
> - Phase 0(Ideation) 신설 — 왈도+히스 병렬 투입
> - 히스 이중 모드 도입 (영감 모드 / 고증 모드)
> - 페이서-매크로 Phase V → Phase I 이동
> - 하트+보이스 → 캐리(Carry)로 통합
> - 퍼스에서 프로포잘 검증 역할 분리 → 레드/키퍼 위임
> - 시스를 Phase 간 게이트키퍼로 명시

## 표준 투입 순서

```
Phase 0 — 발상 (Ideation)
  왈도 · 히스(영감 모드) ─── 병렬

        ↓

Phase I — 설계 (Design)
  포지 · 아크 · 페이서-매크로 ─── 포지→아크 순차, 페이서-매크로 병렬
  히스(고증 모드) ─── 포지가 필요 시 온콜 호출

        ↓ 시스 gate check

Phase II — 집필 (Writing)
  제니

        ↓ 시스 gate check

Phase III — 검증 (Verification)
  레드 · 키퍼 · 퍼스 ─── 병렬

        ↓ 시스 gate check

Phase IV — 감성 + 마무리 (Emotion + Polish)
  캐리 · 리더 ─── 병렬
  페이서-마이크로 → 쿳시 ─── 순차
```

## Phase별 상세

### Phase 0 — 발상 (Ideation)
Beat 설계 전 아이디어 발산 단계. 왈도가 서사적 가능성을 자유롭게 탐색하고, 히스가 **영감 모드**로 역사적 유사 사례와 신화적 모티프를 병렬 제공한다.

- **왈도**: 복선·반전·씨앗 아이디어, 대안 루트 제안
- **히스(영감 모드)**: 왈도의 아이디어에 역사적 평행 사례 첨부. 예) 용병단 내부 반란 아이디어 → 백년전쟁 자유 용병단(Free Companies) 사례 제공

**투입 조건**: 새 챕터 또는 주요 서사 분기점 시작 시
**산출물**: 아이디어 제안서 + 역사 레퍼런스 첨부

### Phase I — 설계 (Design)
포지가 세계관 설정을 확인/갱신하고, 아크가 Beat Sheet를 설계한다. 페이서-매크로가 비트 시트 단계에서 전체 긴장 곡선을 사전 점검한다.

- **포지**: 엔티티 생성/갱신, SSOT 유지
- **아크**: 포지 문서 기반 챕터 구조·Beat 분해
- **페이서-매크로**: 긴장 곡선·정보 배분·복선 설치/회수 전략 점검
- **히스(고증 모드)**: 포지가 새 세력/제도/장소 설정 시 온콜 호출. 고증 검증 및 역사적 배경 보강

**투입 조건**: Phase 0 산출물 작가 승인 후
**산출물**: 갱신된 엔티티 파일 + Beat Sheet + 매크로 페이싱 보고서

### Phase II — 집필 (Writing)
제니가 확정된 Beat Sheet를 산문으로 변환한다.

- **제니**: 포지·아크 문서를 컨텍스트로 투입. 오감 묘사, 심리 묘사, 톤앤매너 준수

**투입 조건**: 시스 gate check 통과 (Beat Sheet 확정, pending 엔티티 해소)
**산출물**: 초안 원고

### Phase III — 검증 (Verification)
세 에이전트가 병렬로 초안을 검증한다.

- **레드**: 논리 결함·클리셰·프로포잘 검증
- **키퍼**: 설정 일관성·고유명사 불일치·참조 무결성·프로포잘 설정 정합성
- **퍼스**: 다른 캐릭터 시점 재해석·시점 편향 감지

**투입 조건**: 시스 gate check 통과 (초안 완성, summary.md 갱신)
**산출물**: 비판 보고서 + 일관성 검증 보고서 + 확장 제안서

### Phase IV — 감성 + 마무리 (Emotion + Polish)
감정·독자 반응 분석과 리듬·교정을 수행한다.

- **캐리**: 감정 곡선 분석 + 캐릭터 말투 일관성을 단일 보고서로 출력
- **리더**: 독자 몰입도·이해도·기대감 평가
- **페이서-마이크로**: 씬 내 템포·대화/묘사 비율·문장 리듬 분석
- **쿳시**: 페이서-마이크로 분석 반영 후 문장 수준 최종 교정

**투입 조건**: 시스 gate check 통과 (검증 보고서 반영, 심각(🔴) 이슈 해소)
**산출물**: 캐릭터 검증 보고서, 독자 반응 보고서, 마이크로 페이싱 보고서, 교정 보고서

## 시스 게이트 체크 프로토콜

매 Phase 전환 시 시스가 다음을 점검한다:

1. **pending 엔티티**: [미생성] 태그가 붙은 항목이 하류 Phase에서 필요한지 확인
2. **참조 무결성**: 문서 간 상호 참조가 유효한지 검사
3. **INDEX.md 동기화**: 새로 생성/수정된 파일이 INDEX에 반영됐는지 확인
4. **이전 Phase 산출물 완결**: 필수 산출물이 모두 존재하는지 확인
5. **심각 이슈 잔존**: 🔴 심각 등급 이슈가 미해결 상태인지 확인

**통과 조건**: 위 5개 항목 중 블로커 없음
**실패 시**: 시스가 작가에게 미해결 항목 목록을 보고하고, 해소 후 재점검

## 히스 이중 모드 운용 규칙

| 모드 | 투입 시점 | 호출 주체 | 목적 | 출력 형식 |
|------|---------|---------|------|---------|
| **영감 모드** | Phase 0 | 파이프라인 (왈도와 병렬) | 아이디어에 역사적 유사 사례·신화 모티프 제공 | 역사적 평행 + 신화적 층위 섹션 |
| **고증 모드** | Phase I | 포지 (온콜) | 새 설정의 시대적 정합성 검증 | 고증 보고서 (정합성 판정 포함) |

프롬프트 투입 시 모드를 명시하여 출력 성격을 분리한다:
- 영감 모드: `[MODE: INSPIRATION]` 접두어
- 고증 모드: `[MODE: VERIFICATION]` 접두어

## 축약 워크플로우

모든 작업이 전체 파이프라인을 거칠 필요는 없다.

| 작업 유형 | 투입 에이전트 |
|----------|------------|
| 브레인덤프 정리 | 시스 → 포지 → 키퍼 |
| 세계관 설정 추가 | 포지 → 히스(고증) → 키퍼 |
| 새 챕터 집필 (전체) | 왈도·히스(영감) → 아크·페이서매크로 → 제니 → 레드·키퍼·퍼스 → 캐리·리더 → 페이서마이크로·쿳시 |
| 기존 챕터 수정 | 제니 → 키퍼 → 쿳시 |
| 캐릭터 신규 생성 | 포지 → 키퍼 |
| 일관성 전체 점검 | 시스 → 키퍼 |
| 아이디어 발산 | 왈도 · 히스(영감) → 포지 |
| 페이싱 점검만 | 페이서-매크로 (전체) + 페이서-마이크로 (개별 씬) |
